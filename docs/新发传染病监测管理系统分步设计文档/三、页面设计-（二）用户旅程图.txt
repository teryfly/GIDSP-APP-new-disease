## （二）、用户旅程图 (User Journey Maps)

### 概述

基于Stage 1的用户故事，将创建关键业务流程的用户旅程图。旅程图展示用户从目标产生到目标达成的完整过程，包括用户步骤、系统响应、情绪变化和痛点识别。

---

### UJ-2.1 个案管理完整旅程

#### 旅程概述

**用户角色：** 疾控业务人员  
**业务目标：** 从接到病例报告到完成个案管理的全流程  
**关联用户故事：** US-3.1, US-3.2, US-3.3, US-3.4, US-3.5, US-3.6, US-3.7  
**预计时长：** 数天到数周（取决于病例复杂度）

---

#### 旅程阶段详细描述

##### 阶段1：接收病例报告

**用户目标：** 快速获取病例信息并评估紧急程度

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 接到医院/社区报告电话<br>2. 登录系统查看是否有相关预警<br>3. 初步判断病例类型（已知疾病/不明原因） |
| **系统响应** | 1. SSO自动认证登录<br>2. 显示待处理预警列表<br>3. 提供快速搜索功能 |
| **用户情绪** | 😐 中性 → 🤔 专注（需要快速判断）                              |
| **痛点**     | - 电话信息不完整<br>- 需要在多个系统间切换查询历史<br>- 无法快速判断是否为重复病例 |
| **机会点**   | - 提供智能病例查重功能<br>- 集成多渠道报告入口（电话、短信、移动端）<br>- 预警信息与病例报告关联提示 |

---

##### 阶段2：创建个案调查表

**用户目标：** 准确录入病例基本信息，建立个案档案

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 点击"新增个案"<br>2. 选择疾病类型<br>3. 填写患者基本信息（姓名、身份证、联系方式）<br>4. 填写流行病学信息（暴露史、接触史、旅行史）<br>5. 填写初步诊断<br>6. 保存并生成个案编号 |
| **系统响应** | 1. 展示表单（根据疾病类型动态调整字段）<br>2. 实时验证必填字段<br>3. 自动检测重复病例（身份证号+疾病）<br>4. 生成唯一个案编号<br>5. 提示保存成功 |
| **用户情绪** | 😐 中性 → 😊 满意（表单清晰易用）<br>或 😤 烦躁（字段过多、验证频繁） |
| **痛点**     | - 表单字段过多，填写耗时<br>- 流行病学信息需要反复询问患者<br>- 患者记忆模糊，信息不准确<br>- 重复病例检测不够智能（仅比对身份证） |
| **机会点**   | - 表单分步填写，支持保存草稿<br>- 智能表单：根据疾病类型显示相关字段<br>- 提供常用信息模板<br>- 支持语音输入<br>- 增强重复检测：模糊匹配姓名+出生日期 |

---

##### 阶段3：安排样本检测

**用户目标：** 及时送检样本，获取实验室确诊

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 进入个案详情页<br>2. 点击"新增检测记录"<br>3. 填写采样信息（时间、样本类型、检测类型）<br>4. 选择送检机构<br>5. 点击"推送至实验室"（如已对接）<br>6. 等待检测结果<br>7. 收到结果后录入或确认 |
| **系统响应** | 1. 展示检测表单<br>2. 验证采样时间合理性<br>3. 推送检测需求至实验室系统<br>4. 记录推送状态<br>5. 接收实验室回传结果（预留功能）<br>6. 更新检测状态为"已确认" |
| **用户情绪** | 😐 中性 → ⏳ 等待焦虑（等待结果）<br>→ 😊 满意（结果及时返回）<br>或 😤 烦躁（结果延迟、需手工录入） |
| **痛点**     | - 当前版本需手工录入检测结果<br>- 实验室系统未对接，信息不同步<br>- 检测结果延迟，影响诊断<br>- 纸质报告需人工录入，易出错 |
| **机会点**   | - 优先对接实验室系统，自动回传结果<br>- 提供检测进度跟踪<br>- 支持扫描或拍照上传检测报告<br>- OCR识别报告内容自动填充 |

---

##### 阶段4：开展流行病学调查

**用户目标：** 追溯传染源，筛查密切接触者

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 进入个案详情页<br>2. 点击"新增追踪记录"<br>3. 选择追踪类型（旅居史/接触史/暴露史）<br>4. 填写时间范围和地点<br>5. 记录接触人员信息<br>6. 进行风险评估<br>7. 点击"推送至流调系统" |
| **系统响应** | 1. 展示追踪表单<br>2. 关联地区信息（支持地图选点）<br>3. 验证时间合理性<br>4. 保存追踪记录<br>5. 推送至流调系统<br>6. 在地图上展示追踪轨迹 |
| **用户情绪** | 🤔 专注（需要详细询问）<br>→ 😓 疲惫（信息收集工作量大）<br>→ 😊 满意（系统辅助整理信息） |
| **痛点**     | - 患者记忆不清，信息碎片化<br>- 地点描述不准确，难以定位<br>- 接触人员信息不完整<br>- 需要多次补充和修正<br>- 手工绘制轨迹图耗时 |
| **机会点**   | - 支持地图选点记录地理位置<br>- 集成手机信令数据（需授权）<br>- 提供时间轴视图辅助回忆<br>- 自动生成轨迹图和接触关系图<br>- 支持批量导入密接者信息 |

---

##### 阶段5：治疗与随访管理

**用户目标：** 持续跟踪患者健康状况，评估治疗效果

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 定期进入个案详情页<br>2. 点击"新增随访记录"<br>3. 选择随访方式（电话/现场/线上）<br>4. 记录健康状态、体温、症状<br>5. 评估治疗依从性<br>6. 计划下次随访时间<br>7. 必要时添加治疗记录 |
| **系统响应** | 1. 展示随访表单（预填上次随访信息）<br>2. 自动计算随访间隔<br>3. 保存随访记录<br>4. 设置随访提醒<br>5. 更新个案状态<br>6. 生成健康趋势图 |
| **用户情绪** | 😐 中性 → 😊 满意（患者好转）<br>或 😟 担忧（病情恶化）<br>→ ⏰ 提醒焦虑（随访任务多） |
| **痛点**     | - 随访任务多，容易遗漏<br>- 电话随访患者不配合<br>- 信息记录分散，难以看到整体趋势<br>- 需要手工计算随访频率 |
| **机会点**   | - 智能随访提醒（根据病情自动调整频率）<br>- 提供随访话术模板<br>- 可视化健康趋势图<br>- 支持患者自助上报（移动端）<br>- 批量随访任务管理 |

---

##### 阶段6：结案与归档

**用户目标：** 完成病例管理，归档病例资料

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 确认患者已康复或达到结案条件<br>2. 进入个案详情页<br>3. 检查所有记录完整性<br>4. 点击"结案"<br>5. 选择结案原因（治愈/死亡/终止跟踪）<br>6. 导出完整病例档案<br>7. 归档保存 |
| **系统响应** | 1. 检查必要记录完整性<br>2. 更新个案状态为"已关闭"<br>3. 生成完整病例报告（PDF）<br>4. 记录结案操作日志<br>5. 触发数据归档流程 |
| **用户情绪** | 😊 满意（完成任务）<br>→ 😌 轻松（工作负担减轻）               |
| **痛点**     | - 不清楚结案标准<br>- 记录不完整导致无法结案<br>- 导出报告格式不符合要求<br>- 归档流程繁琐 |
| **机会点**   | - 提供结案检查清单<br>- 自动检测记录完整性<br>- 提供多种报告模板<br>- 支持批量结案<br>- 自动归档至文档管理系统 |

---

### UJ-2.2 不明原因病例确诊转换旅程

#### 旅程概述

**用户角色：** 疾控业务人员  
**业务目标：** 从发现不明原因病例到确诊并转入个案管理  
**关联用户故事：** US-4.1, US-4.2, US-4.3  
**预计时长：** 数天到数周（取决于检测进度）

---

#### 旅程阶段详细描述

##### 阶段1：发现不明原因病例

**用户目标：** 识别异常病例，建立初步档案

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 接到医院报告：患者症状不符合已知疾病<br>2. 登录系统<br>3. 判断是否为不明原因病例<br>4. 点击"新增不明原因病例"<br>5. 填写患者信息和临床症状<br>6. 记录疑似病原体<br>7. 保存记录 |
| **系统响应** | 1. 展示不明原因病例表单<br>2. 生成不明原因病例编号<br>3. 初始状态设为"待检测"<br>4. 提示保存成功 |
| **用户情绪** | 🤔 疑惑（症状不明确）<br>→ 😟 担忧（可能是新发传染病）<br>→ 😐 专注（需要详细记录） |
| **痛点**     | - 难以判断是否为不明原因病例<br>- 缺乏参考标准和案例库<br>- 症状描述不规范 |
| **机会点**   | - 提供不明原因病例判定指南<br>- 提供症状标准化词库<br>- 智能提示类似历史案例 |

---

##### 阶段2：紧急上报与样本送检

**用户目标：** 及时上报并尽快确认病原体

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 进入不明原因病例详情<br>2. 点击"上报应急系统"<br>3. 确认上报信息<br>4. 点击"推送流调系统"<br>5. 安排样本采集<br>6. 点击"新增检测记录"<br>7. 填写采样信息<br>8. 送检至实验室 |
| **系统响应** | 1. 推送至应急指挥系统<br>2. 记录推送状态和时间<br>3. 推送至流调系统<br>4. 创建检测记录<br>5. 更新病例状态为"检测中" |
| **用户情绪** | 😟 担忧（可能是重大疫情）<br>→ ⏳ 焦急（等待检测结果）<br>→ 😓 压力（需要及时上报） |
| **痛点**     | - 上报流程不清晰<br>- 需要在多个系统重复录入<br>- 检测周期长，等待焦虑<br>- 缺乏检测进度跟踪 |
| **机会点**   | - 一键上报多个系统<br>- 自动同步基本信息<br>- 提供检测进度实时跟踪<br>- 紧急检测绿色通道标识 |

---

##### 阶段3：等待与跟进检测结果

**用户目标：** 及时获取检测结果，补充检测信息

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 定期查看检测记录<br>2. 联系实验室询问进度<br>3. 收到初步结果后录入系统<br>4. 必要时补充检测项目<br>5. 等待最终确诊结果 |
| **系统响应** | 1. 展示检测记录列表<br>2. 显示检测状态（待确认/检测中）<br>3. 接收手工录入的检测结果<br>4. 更新检测状态 |
| **用户情绪** | ⏳ 焦急等待<br>→ 😟 担忧（结果不明朗）<br>→ 😓 疲惫（反复跟进） |
| **痛点**     | - 实验室系统未对接，无法自动获取结果<br>- 检测进度不透明<br>- 需要反复电话沟通<br>- 纸质报告需手工录入 |
| **机会点**   | - 对接实验室系统，自动推送结果<br>- 提供检测进度看板<br>- 短信/邮件提醒检测结果<br>- 支持报告扫描上传和OCR识别 |

---

##### 阶段4：确认病原体并数据校验

**用户目标：** 确认疾病类型，准备转入个案管理

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 收到确诊检测结果<br>2. 进入不明原因病例详情<br>3. 录入或确认检测结果<br>4. 系统自动更新状态为"已确诊"<br>5. 确认确诊疾病名称<br>6. 检查是否有重复病例 |
| **系统响应** | 1. 接收检测结果<br>2. 更新病例状态为"已确诊"<br>3. 自动执行数据校验（检查重复）<br>4. 提示"可推送至个案管理"<br>5. 如发现重复，提示用户处理 |
| **用户情绪** | 😊 松了口气（终于确诊）<br>→ 😐 专注（准备转换流程）<br>或 😤 烦躁（发现重复病例） |
| **痛点**     | - 重复病例检测不够智能<br>- 不清楚如何处理重复病例<br>- 担心数据丢失或冲突 |
| **机会点**   | - 智能重复检测（模糊匹配）<br>- 提供重复病例处理指引<br>- 支持病例合并功能 |

---

##### 阶段5：推送至个案管理

**用户目标：** 将确诊病例转入标准个案管理流程

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 点击"病例推送"按钮<br>2. 确认推送对话框<br>3. 等待系统处理<br>4. 查看推送结果<br>5. 进入个案管理模块查看新生成的个案 |
| **系统响应** | 1. 弹出确认对话框<br>2. 封装病例数据<br>3. 创建个案记录（数据映射）<br>4. 复制检测记录<br>5. 推送至应急系统和流调系统<br>6. 更新不明原因病例状态为"已推送"<br>7. 记录推送日志<br>8. 提示推送成功并显示新个案编号 |
| **用户情绪** | 😐 专注（等待处理）<br>→ 😊 满意（推送成功）<br>→ 😌 轻松（进入标准流程） |
| **痛点**     | - 推送过程不透明<br>- 推送失败时不知如何处理<br>- 需要在两个模块间切换查看 |
| **机会点**   | - 显示推送进度条<br>- 推送失败自动重试<br>- 推送成功后直接跳转至新个案<br>- 提供推送日志查询 |

---

##### 阶段6：后续个案管理

**用户目标：** 在个案管理模块继续跟进病例

| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **用户行为** | 1. 进入个案管理模块<br>2. 查看新生成的个案<br>3. 继续添加随访、治疗、追踪记录<br>4. 按标准个案管理流程处理 |
| **系统响应** | 1. 展示个案详情<br>2. 显示个案来源为"不明原因转入"<br>3. 包含原有检测记录<br>4. 支持后续管理操作 |
| **用户情绪** | 😊 满意（数据完整保留）<br>→ 😐 中性（进入常规工作）           |
| **痛点**     | - 不明原因病例的历史信息查询不便<br>- 两个模块的数据关联不明显 |
| **机会点**   | - 在个案详情中提供"查看原不明病例"链接<br>- 时间轴视图展示完整病例历程 |

| **数据孤岛**   | 两个模块数据关联不明显       | 低       |

#### UJ-2.3 预警响应与处理旅程

##### 旅程概述

**用户角色：** 疾控业务人员  
**业务目标：** 从接收预警消息到完成预警处理  
**关联用户故事：** US-2.1, US-3.1  
**预计时长：** 数小时到数天

---

##### 旅程阶段详细描述

##### 阶段1：接收预警通知
