（五）、页面字段定义表 (Page Field Definition Tables)
概述
基于Stage 4的低保真原型，将详细定义每个页面的所有输入/输出字段，并映射到后端数据模型（ER图）的实体属性。

PF-5.1 页面分组计划
页面分组策略
分组编号	分组名称	包含页面	优先级
Group A	个案管理核心页面	个案列表、个案详情、新增个案、随访/治疗/检测/追踪表单	P0
Group B	不明原因疾病管理	病例列表、病例详情、新增病例、推送确认	P0
Group C	预警管理页面	预警列表、预警详情、处理表单	P0
Group D	统计分析页面	疾病统计、病原体统计、查询条件、导出	P1
Group E	基础数据管理	疾病编码、病原体、组织机构、地区	P1
Group F	系统管理页面	用户管理、系统配置、接口监控、日志	P1
Group G	通用组件	首页工作台、消息中心、个人中心	P0

格式验证			
身份证号	idCardFormat	18位身份证号格式	idCardNo
手机号	phoneFormat	11位手机号格式	phone
邮箱	emailFormat	邮箱格式	email
日期	dateFormat	YYYY-MM-DD	所有日期字段
日期时间	dateTimeFormat	YYYY-MM-DD HH:mm:ss	所有日期时间字段
长度验证			
短文本	shortText	2-50字符	姓名、编码类字段
中文本	mediumText	2-200字符	标题、名称类字段
长文本	longText	10-1000字符	描述、详情类字段
范围验证			
年龄	ageRange	0-150	age
体温	temperatureRange	35.0-42.0	temperature
百分比	percentageRange	0-100	各种百分比字段
唯一性验证			
编码唯一	codeUnique	数据库唯一性检查	caseNo, diseaseCode, pathogenCode等
身份证唯一	idCardUnique	同一疾病不重复	idCardNo (个案)
逻辑验证			
日期先后	dateSequence	开始日期 ≤ 结束日期	所有日期范围字段
日期不晚于今天	dateNotFuture	日期 ≤ 今天	reportDate, symptomOnsetDate等
日期不早于基准	dateNotBefore	诊断日期 ≥ 症状开始日期	diagnosisDate
必填验证			
必填	required	不能为空	所有标注"是否必填=是"的字段
条件必填	conditionalRequired	满足条件时必填	如选择"是"后的详情字段
PF-5.11 字段状态与权限控制
字段编辑权限矩阵
字段类型	新增时	编辑时	详情查看	列表查看	说明
系统生成字段					
ID类	自动生成	不可编辑	可查看	可查看	如caseId, caseNo
时间戳	自动生成	自动更新	可查看	可查看	如createdAt, updatedAt
操作人	自动填充	不可编辑	可查看	可查看	如createdBy, reportUserId
业务字段					
基本信息	可编辑	可编辑	可查看	可查看	如姓名、性别、年龄
敏感信息	可编辑	可编辑	脱敏显示	脱敏显示	如身份证号、手机号
状态字段	初始值	根据状态机	可查看	可查看	如caseStatus, alertStatus
关联字段					
外键	可选择	可修改	显示关联名称	显示关联名称	如diseaseId → diseaseName
字段显示权限（按角色）
字段敏感度	系统管理员	省级人员	市级人员	县级人员	说明

业务字段	✓	✓	✓	✓	如症状描述、诊断信息
敏感字段	✓	✓ (脱敏)	✓ (脱敏)	✓ (脱敏)	如身份证号、详细地址
管理字段	✓	✓ (只读)	✗	✗	如系统配置、用户管理
